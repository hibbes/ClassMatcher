Bedienungsanleitung – Schiller-Klassen-Mixer

1. Programm starten

Öffnen Sie ein Terminal, wechseln Sie in den Programmordner und führen Sie aus:

./run.sh

Der Browser öffnet sich automatisch auf http://localhost:5001. Das Programm läuft ausschließlich lokal – keine Daten verlassen Ihren Computer.

2. CSV-Datei laden

Ziehen Sie die Anmeldungs-CSV-Datei auf das Ablage-Feld oder klicken Sie darauf und wählen Sie die Datei aus. Das Programm liest alle Schülerdaten ein und führt sofort eine erste automatische Klassenzuweisung durch.

Die CSV-Datei muss das Format der Schulverwaltungssoftware haben (Felder: ID, Name, Vorname, Geschlecht, Profil1, Klassenpartner, Fremdsprache2 …).

3. Freundeswünsche klären ⚠

Das Feld Klassenpartner enthält freie Texteingaben der Eltern. Das Programm versucht, die genannten Namen automatisch den vorhandenen Schülerinnen und Schülern zuzuordnen (Fuzzy-Matching).

Erscheint der orangefarbene ⚠-Button oben, gibt es Fälle, die nicht eindeutig erkannt wurden. Klicken Sie darauf, um jeden unsicheren Treffer manuell zu bestätigen oder zu überspringen. Erst danach ist die Zuweisung vollständig.

4. Klassenübersicht verstehen

Jede Klasse wird als Spalte dargestellt. Auf jeder Schülerkarte sehen Sie:

  • / Geschlecht (blau = Junge, pink = Mädchen)
  • Name des Schülers / der Schülerin
  • Musik Bili Normal gewählter Zug
  • Franzö. Latein zweite Fremdsprache
  • ✓ Name Freundeswunsch erfüllt (grün)
  • ✗ Name → 5z-1 Wunsch nicht erfüllt, mit Zielklasse (orange)
  • 🔒 Schüler wurde manuell verschoben und ist gesperrt

Der Klassenname (z.B. „Musikzug") kann durch Anklicken direkt umbenannt werden.

5. Parameter einstellen

Die linke Seitenleiste enthält Regler, mit denen Sie die Zuweisung steuern. Jede Änderung löst nach kurzer Pause automatisch eine Neuberechnung aus.

Max. Schüler/KlasseMaximale Klassengröße. Daraus ergibt sich auch die Anzahl der Klassen automatisch.
FreundeswunschWie stark soll versucht werden, Freundschaftswünsche zu erfüllen? (0 = ignorieren, 10 = maximale Priorität)
GeschlechterbalanceWie ausgeglichen soll das Verhältnis von Jungen und Mädchen pro Klasse sein?
Musikzug-VerteilungWie streng sollen die Musikzug-Schüler:innen auf genau zwei Klassen (nie die Bili-Klasse) gleichmäßig verteilt werden? 0 % = egal, 100 % = strikte 50/50-Aufteilung.

Feste Regeln, immer aktiv: Die Bili-Klasse bekommt nie Musikzug-Schüler:innen. Die letzte Normalzug-Klasse bleibt latein­frei. Manuell verschobene Schüler:innen können diese Regeln überschreiben.

6. „Nicht zusammen"-Paare

Klicken Sie in der Seitenleiste auf + Paar hinzufügen, suchen Sie zwei Schülerinnen oder Schüler und speichern Sie. Das Programm sorgt dafür, dass diese beiden niemals in dieselbe Klasse kommen – bei jeder Neuberechnung.

Eingetragene Paare können mit dem ✕-Symbol wieder entfernt werden.

7. Manuell verschieben

Schülerinnen und Schüler können per Drag & Drop von einer Klasse in eine andere gezogen werden. Das Programm merkt sich diese Entscheidung (🔒) und behält sie auch bei automatischen Neuberechnungen bei.

Das Schloss-Symbol 🔒 auf einer Karte anklicken hebt die Sperre für diesen Schüler auf. Mit dem Button „Manuelle Verschiebungen aufheben" werden alle Sperren auf einmal gelöscht – bei der nächsten Neuberechnung verteilt der Algorithmus wieder frei.

8. Drucken

Der Button Drucken öffnet den Druckdialog des Browsers. Für jede Klasse wird eine eigene Seite erzeugt mit:

  • Schullogo, Klassenname und Datum
  • Statistik-Zeile (Anzahl, Jungen/Mädchen, erfüllte Wünsche)
  • Tabelle mit: Lfd. Nr. · Name, Vorname · Zug · 2. Fremdsprache · Wunschfreunde in der Klasse
Tipp: Im Browser-Druckdialog „Hintergrundgrafiken drucken" aktivieren, damit die farbige Tabellenüberschrift erscheint.

9. Häufige Fragen

Warum sind Musikzug und Bili-Klasse größer als der Normalzug? Musik- und Bili-Klassen werden automatisch mit Normalzug-Schülern aufgefüllt, damit alle Klassen ähnlich groß sind. Welche Normalzug-Schüler wohin kommen, optimiert der Algorithmus.
Warum ändert sich das Ergebnis bei jedem Klick auf „Neu zuweisen"? Der Algorithmus (Simulated Annealing) startet mit einer zufälligen Ausgangsbelegung. Das Ergebnis kann leicht variieren, erfüllt aber stets die eingestellten Kriterien optimal.
Was passiert, wenn ein Freundeswunsch nicht erfüllbar ist? Das Programm zeigt auf der Karte den Namen und die Klasse, in der der gewünschte Freund gelandet ist (orangefarbener Tag). Es handelt sich um einen Hinweis, keine Fehlermeldung.
Kann ich die Klassen umbenennen? Ja – einfach den Klassennamen in der Spaltenüberschrift anklicken, neuen Namen eintippen und mit Enter bestätigen.
Werden meine Daten gespeichert? Nur wenn Sie es aktiv tun. Mit dem Button ↓ Speichern (oben rechts, nach der Zuweisung) laden Sie den aktuellen Stand als JSON-Datei herunter – inklusive aller manuellen Verschiebungen, Wunschauflösungen und Klassenumbenennungen. Diese Datei können Sie später über ↑ Gespeicherten Stand laden im Upload-Bereich wiederherstellen. Ohne Speichern gehen alle Daten beim Beenden verloren.
